# 快速提交 Vercel 修复 - 两种方法

## 🚀 方法1：双击批处理文件（最简单）⭐

### 步骤：

1. **找到文件**
   - 在 `工具脚本` 文件夹中找到：`提交Vercel修复.bat`

2. **双击运行**
   - 直接双击 `提交Vercel修复.bat`
   - 会自动执行所有步骤

3. **等待完成**
   - 脚本会自动：
     - 添加所有修改的文件
     - 提交代码
     - 推送到 GitHub
   - 完成后会显示结果

4. **完成！**
   - Vercel 会自动检测到新提交
   - 等待 1-2 分钟后访问你的 Vercel URL

## 💻 方法2：使用 CMD（命令行）

### 步骤：

1. **打开 CMD**
   - 按 `Win + R`
   - 输入 `cmd` 并回车

2. **切换到项目目录**
   ```cmd
   cd /d "C:\Users\温柔的男子啊\Desktop\crusor\圆心工作\工具和脚本\工具脚本"
   ```

3. **执行 Git 命令**
   ```cmd
   git add .
   git commit -m "Fix Vercel deployment: encoding, paths, and dependencies"
   git push
   ```

4. **等待完成**
   - 如果成功，会显示推送成功的信息
   - Vercel 会自动重新部署

## ⚠️ 如果遇到问题

### 问题1：Git 未安装

**错误信息**：`'git' 不是内部或外部命令`

**解决方法**：
1. 下载 Git：https://git-scm.com/download/win
2. 安装时选择 "Add Git to PATH"
3. 重新打开 CMD 或批处理文件

### 问题2：GitHub Token 未设置

**错误信息**：`remote: Repository not found` 或 `Authentication failed`

**解决方法**：
1. 检查环境变量 `GITHUB_TOKEN` 是否已设置
2. 如果没有，运行：
   ```cmd
   setx GITHUB_TOKEN "你的token"
   ```
3. 重新打开 CMD 窗口

### 问题3：没有更改需要提交

**错误信息**：`nothing to commit`

**解决方法**：
- 这是正常的，说明文件已经是最新的
- 可以直接推送到 GitHub（如果有未推送的提交）
