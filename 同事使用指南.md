# 同事使用应用指南

## 📋 方案选择

根据你的需求，有三种方式让同事使用应用：

### 方案1：在线访问（推荐）⭐
- **优点**：无需安装，打开浏览器即可使用
- **适用**：所有同事，特别是非技术人员
- **需要**：将应用部署到 Vercel（免费）

### 方案2：本地运行
- **优点**：数据存储在本地，更安全
- **适用**：需要离线使用或数据敏感的同事
- **需要**：安装 Python 和依赖

### 方案3：GitHub 仓库访问
- **优点**：可以查看代码，学习使用
- **适用**：技术人员，想了解代码的同事
- **需要**：GitHub 账号

---

## 🚀 方案1：在线访问（推荐）

### 步骤1：部署到 Vercel（如果还没部署）

#### 1.1 访问 Vercel
1. 打开浏览器，访问：https://vercel.com
2. 点击 "Sign Up" 或 "Log In"
3. 选择 "Continue with GitHub"（使用你的 GitHub 账号登录）

#### 1.2 导入项目
1. 登录后，点击 "Add New Project"
2. 在 "Import Git Repository" 中，选择你的仓库：`mashitan1111/todo-list-app`
3. 点击 "Import"

#### 1.3 配置项目
Vercel 会自动检测配置，通常不需要修改：
- **Framework Preset**: Other
- **Root Directory**: `./`（或留空）
- **Build Command**: （留空）
- **Output Directory**: （留空）

#### 1.4 部署
1. 点击 "Deploy"
2. 等待部署完成（通常 1-2 分钟）
3. 部署成功后，你会看到一个 URL，例如：`https://todo-list-app-xxx.vercel.app`

### 步骤2：分享给同事

#### 方式A：直接分享 URL
```
将 Vercel 部署后的 URL 分享给同事：
https://your-project.vercel.app
```

#### 方式B：设置自定义域名（可选）
1. 在 Vercel 项目设置中，点击 "Domains"
2. 添加你的自定义域名（如果有）
3. 按照提示配置 DNS

### 步骤3：同事访问
1. 同事打开浏览器
2. 输入你分享的 URL
3. 即可开始使用应用

## 💻 方案2：本地运行

### 步骤1：分享仓库地址

告诉同事你的 GitHub 仓库地址：
```
https://github.com/mashitan1111/todo-list-app
```

### 步骤2：同事克隆仓库

#### 方法A：使用 Git（推荐）
```bash
# 1. 安装 Git（如果未安装）
# 下载：https://git-scm.com/download/win

# 2. 克隆仓库
git clone https://github.com/mashitan1111/todo-list-app.git

# 3. 进入目录
cd todo-list-app
```

#### 方法B：直接下载 ZIP
1. 访问：https://github.com/mashitan1111/todo-list-app
2. 点击绿色的 "Code" 按钮
3. 选择 "Download ZIP"
4. 解压到本地目录

### 步骤3：安装依赖

```bash
# 1. 确保已安装 Python 3.7+
python --version

# 2. 安装依赖
pip install -r requirements.txt
```

### 步骤4：运行应用

#### Windows 用户：
```bash
# 方法1：双击运行
启动工作待办清单.bat

# 方法2：命令行运行
python 工作待办清单桌面应用_精美版.py
```

#### Mac/Linux 用户：
```bash
python3 工作待办清单桌面应用_精美版.py
```

### 步骤5：访问应用

应用启动后，浏览器会自动打开，或手动访问：
```
http://127.0.0.1:5000
```

## 📖 方案3：GitHub 仓库访问

### 步骤1：设置仓库为公开（如果需要）

如果想让所有同事都能访问代码：

1. 访问：https://github.com/mashitan1111/todo-list-app/settings
2. 滚动到页面底部
3. 在 "Danger Zone" 中，点击 "Change visibility"
4. 选择 "Make public"（或保持 private，只邀请特定同事）

### 步骤2：邀请协作者（如果仓库是私有的）

1. 访问：https://github.com/mashitan1111/todo-list-app/settings/access
2. 点击 "Invite a collaborator"
3. 输入同事的 GitHub 用户名或邮箱
4. 选择权限级别（通常选择 "Write"）
5. 发送邀请

### 步骤3：同事访问

同事收到邀请后：
1. 登录 GitHub 账号
2. 接受邀请
3. 可以查看代码、提交问题、提出建议等

## 🔐 权限管理

### 如果使用在线版本（Vercel）

**当前状态**：所有访问同一 URL 的人共享同一个数据库

**如果需要多用户隔离**：
1. 需要添加用户认证功能
2. 或为每个部门/团队部署独立的实例

### 如果使用本地版本

**缺点**：需要各自维护和更新

## ❓ 常见问题

### Q1: 同事访问在线版本时，数据会混乱吗？

**A**: 是的，当前版本所有用户共享同一个数据库。如果需要隔离，需要：
- 添加用户登录功能
- 或为每个团队部署独立实例

### Q2: 如何更新应用？

**在线版本**：
- 你更新代码并推送到 GitHub
- Vercel 会自动重新部署
- 同事刷新页面即可看到更新

**本地版本**：
- 同事需要重新克隆或下载最新代码
- 或使用 `git pull` 更新

### Q3: 数据会丢失吗？

**在线版本（Vercel）**：
- ⚠️ **重要**：Vercel 使用临时文件系统
- 数据可能在函数重启后丢失
- **建议**：使用外部数据库（Supabase、PlanetScale 等）

**本地版本**：
- 数据存储在本地 SQLite 数据库
- 只要不删除数据库文件，数据不会丢失

### Q4: 如何备份数据？

**在线版本**：
- 定期导出数据
- 或配置自动备份到云存储

**本地版本**：
- 备份 `tasks.db` 文件
- 或定期导出为 JSON/CSV
