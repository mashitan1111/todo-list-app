# 工作待办清单桌面应用使用说明

## 【元数据】
- **创建日期**：2026-01-04
- **版本**：V1.0 精美版
- **用途**：可视化显示工作待办清单，支持任务管理和状态同步

---

## 功能特点

### ✨ 精美界面设计
- **现代化UI**：渐变背景、毛玻璃效果、卡片式布局
- **清晰视觉层次**：颜色区分优先级、图标提示、分组展示
- **流畅交互体验**：动画过渡、悬停效果、状态反馈
- **自然信息流**：优先级排序、智能分组、进度可视化

### 📊 核心功能
1. **自动读取任务**：从`工作待办清单.md`读取所有任务
2. **智能分类**：按优先级（紧急/高/普通）自动分组
3. **状态管理**：点击任务可标记完成/未完成
4. **推荐工作**：自动显示`推荐改变清单.md`中的推荐工作
5. **进度统计**：实时显示完成进度和任务统计
6. **状态同步**：任务状态保存在`任务状态.json`中

## 界面说明

### 头部区域
- **标题**：工作待办清单
- **更新时间**：显示最后刷新时间
- **统计卡片**：
  - 🚨 紧急任务数量
  - 📝 待完成任务总数

- **待完成任务**（左侧）：
  - 🚨 紧急任务（红色边框）
  - ⚠️ 高优先级任务（黄色边框）
  - 📋 普通任务（青色边框）
- **已完成任务**（右侧）：
  - 显示最近完成的10个任务
  - 灰色显示，带删除线

### 推荐工作区域
- 显示来自`推荐改变清单.md`的推荐工作
- 包含推荐编号、日期、内容和理由

## 操作说明

### 完成任务
1. 点击待完成任务项（整个卡片区域）
2. 任务会自动标记为完成
3. 页面自动刷新，任务移到"已完成"区域

### 取消完成
1. 点击已完成任务项
2. 任务会重新标记为未完成
3. 页面自动刷新，任务回到"待完成"区域

### 刷新数据
- 点击"🔄 刷新数据"按钮
- 重新读取Markdown文件
- 更新所有任务和推荐工作

## 优先级识别

应用会自动识别任务优先级：

- **【紧急】或【P0】** → 🚨 紧急任务（红色）
- **【P1】** → ⚠️ 高优先级（黄色）
- **其他** → 📋 普通任务（青色）

## 任务来源显示

如果任务中包含"来源："标记，会在任务下方显示来源信息。

例如：
```
- [ ] 更新客户画像 - 来源：RAG知识库/...
```
会显示为：
```
更新客户画像
📍 RAG知识库/...
```

## 常见问题

### Q1: 浏览器没有自动打开？
**A**: 等待1-2秒，如果还没打开，手动访问：`http://127.0.0.1:5000`

### Q2: 提示"无法启动应用"？
**A**: 
1. 检查是否安装了Python
2. 检查是否安装了Flask：`pip install flask`
3. 检查文件路径是否正确

### Q3: 任务状态没有保存？
**A**: 
1. 检查`任务状态.json`文件是否创建
2. 检查文件写入权限
3. 查看控制台是否有错误信息

### Q4: 读取不到任务？
**A**: 
1. 检查`工作待办清单.md`文件是否存在
2. 检查文件路径是否正确
3. 确保任务格式为：`- [ ] 任务内容`或`- [x] 任务内容`

### Q5: 推荐工作没有显示？
**A**: 
1. 检查`推荐改变清单.md`文件是否存在
2. 检查文件格式是否正确
3. 确保推荐记录格式为：`#### [日期] 推荐改变 #编号`
